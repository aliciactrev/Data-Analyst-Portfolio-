
## About Data Mart SQL Project 
The objective of this case study is to analyse weekly sales data for a fictional business, Data Mart, to assess the impact of a major operational change — the introduction of sustainable packaging in June 2020.

The analysis focuses on data cleaning, transformation, aggregation, and comparative analysis to support business decision-making.

Case study link: https://8weeksqlchallenge.com/case-study-5/

---

### SQL Technical Skills Applied

- Data cleaning and transformation (date handling, derived columns, null management)
- Aggregation and grouping across regions, platforms, and customer segments
- Before-and-after analysis to measure business impact
- Use of CTEs and structured SQL for readable, maintainable queries
- Translating data outputs into clear business insights

---

### Key Questions Answered

- How did sales performance change after the packaging update?
- Which regions, platforms, and customer segments were most affected?
- What trends emerged before and after the change?

## About the Clique Bait SQL Project

## Overview

The goal of this project is to analyse digital interaction and event data from a fictional online store, Clique Bait, to understand user behaviour and funnel performance. ---

## SQL Technical Skills Applied

- Data cleaning and joining multiple event and reference tables
- Aggregation and grouping to quantify user activity and interactions
- Funnel analysis to calculate conversions, abandonment, and engagement rates
- Use of Common Table Expressions (CTEs) for structured query logic
- Turning raw event data into interpretable business metrics

---

## Key Questions Answered

- How many users and unique visits are present in the data?
- What are the key engagement and conversion rates at different stages of the user funnel?
- Which products and pages drive the most views, cart actions, and purchases?
- What patterns exist in campaign performance and user interactions?
- How can funnel fallout and abandonment be quantified for optimisation?
